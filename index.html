Flat JS
A(nother) JS Framework

What's Different?

Markup Driven

FlatJS uses raw HTML to create templates, and loads model / resource data from your markup. Complete JS control, with the HTML your server is already producing.

Progressive Enhancement

FlatJS employs an old school technique, so you can forget changing how your server is setup. Don't have anxiety over SEO issues.

Limited Boilerplate Code

Having to copy paste code to start a project is bad. FlatJS provides a syntax which strives to reduce boilerplate code while still providing structure.

What's the Same?

MV* Based

FlatJS employs an MV* like scheme, but uses different terminology to avoid confusion with server side paradigms. Models are Resources and views are Components.

Two Way Data Binding

Components read your HTML on intializing and create a JSON object which reflects your template, resource data included. Those object's values are bound to your markup.

OOP From the Ground Up

John Resig's inheritance method provides class based extensions, and FlatJS adds a syntax for calling private member methods, perserving the meaning of this.
